# Decision Procedure

Read the title and main problem/feature statement first; use body, comments,
labels, changed files, and diff only to confirm or disambiguate central
interests. Select the topics whose behavior contract the item changes under
the fixed taxonomy and boundary overlay, applying the deliverable test before
every label.

# Cardinality Rules

Use at most 3 topics, in priority order with the primary changed surface
first. Drop labels that are only delivery mechanisms, producer/symptom
locations, motivations, commenter discussion, paths, examples, or
consequences. When a specific topic applies, add its generic counterpart only
for a separately changed concern. Do not collapse a clearly multi-surface item
to one broad label.
